feat(auth): transition to dynamic entity-based permission system

This commit implements a fundamental architectural shift in the permission system:
- Replaced the static whitelist of tables with a dynamic 'Entity' table.
- Implemented an 'Entities' module for full CRUD management of system resources.
- Updated backend PermissionsGuard to validate against database entities and their
  permission types (CRUD, READ_ONLY, PROCESS).
- Implemented 'Entities' feature on frontend with a dedicated dashboard and creation flow.
- Removed legacy mock features (clients, invoices, purchase-orders).
- Updated agent documentation to reflect the new dynamic model and architectural patterns.

BREAKING CHANGE: The permission system now depends on the 'Entity' table in the database
instead of the 'MANAGED_ENTITIES' constant in the shared package.
